# Expanded Behavior + Typespecs Tiers

Tiers that hash post-macro-expansion BEAM debug info (expanded_behavior) and
`@spec`/`@type` declarations (typespecs).

## Intent

`api_boundary` and `implementation` hash source-level AST. After macro
expansion, `use`-emitted functions and `@spec` declarations live only in beam
debug_info. The `expanded_behavior` tier reads `:beam_lib.chunks(beam,
[:debug_info])`, canonicalizes the expanded AST, and hashes it — this is what
catches drift in functions generated by `use SomeDSL` when the DSL itself
changes. The `typespecs` tier hashes the module's `@spec`/`@type` declarations
independently, so a typespec change produces its own drift finding.
Modules compiled with `@compile {:no_debug_info, true}` degrade gracefully via
`detector_unavailable`.

## Requirements

```yaml spec-requirements
- id: specled.expanded_behavior_tier.reads_beam_debug_info
  statement: >-
    SpecLedEx.Realization.ExpandedBehavior.hash/2 shall read
    `:beam_lib.chunks(beam, [:debug_info])` for the target module and
    extract the post-expansion AST for each declared MFA. The AST is
    canonicalized via SpecLedEx.Realization.Canonical.normalize/1 and
    hashed via the deterministic term_to_binary pipeline.
  priority: must
  stability: evolving
- id: specled.expanded_behavior_tier.hash_stable_on_refactor
  statement: >-
    A cosmetic refactor inside an expanded function body (local
    renames, reflow) shall not change the expanded_behavior hash. The
    canonicalizer applies the same rules to expanded AST as to source
    AST.
  priority: must
  stability: evolving
- id: specled.expanded_behavior_tier.no_debug_info_detector_unavailable
  statement: >-
    When a target module was compiled with
    `@compile {:no_debug_info, true}` (or debug_info is otherwise
    absent), ExpandedBehavior shall emit a `detector_unavailable`
    finding with reason `:debug_info_stripped` naming the module and
    return no hash. The subject is not considered drifted on this
    tier.
  priority: must
  stability: evolving
- id: specled.expanded_behavior_tier.typespecs_hashes_spec_and_type
  statement: >-
    SpecLedEx.Realization.Typespecs.hash/2 shall hash the sorted
    `@spec` and `@type` declarations for each declared MFA/type.
    Sorting is lexicographic on the serialized form to prevent
    declaration-order-induced drift.
  priority: must
  stability: evolving
- id: specled.expanded_behavior_tier.typespecs_drift_finding
  statement: >-
    A change to a declared `@spec` that alters the spec's AST (e.g.,
    adding or removing an argument type) shall emit
    `branch_guard_realization_drift` with tier `:typespecs` naming the
    MFA and subject. Formatting/whitespace changes to the `@spec` shall
    not drift.
  priority: must
  stability: evolving
- id: specled.expanded_behavior_tier.mixed_fixture_coverage
  statement: >-
    The `test/fixtures/sample_project/` fixture shall contain one
    module compiled with `@compile {:no_debug_info, true}` and other
    modules with default debug_info enabled. Integration tests shall
    exercise both paths: the degraded module emits
    `detector_unavailable`, the normal modules produce hashes.
  priority: must
  stability: evolving
```

## Scenarios

```yaml spec-scenarios
- id: specled.expanded_behavior_tier.scenario.expanded_hash_stable_on_rename
  given:
    - a fixture module using `use SomeDSL` that expands into `handle_event/2`
    - "a committed expanded_behavior hash for that MFA"
    - a local-variable rename in the DSL's emitted body (no semantic change)
  when:
    - mix spec.check runs with expanded_behavior enabled
  then:
    - no drift finding references that MFA
  covers:
    - specled.expanded_behavior_tier.reads_beam_debug_info
    - specled.expanded_behavior_tier.hash_stable_on_refactor
- id: specled.expanded_behavior_tier.scenario.no_debug_info_degrades
  given:
    - fixture `NoDebug` compiled with `@compile {:no_debug_info, true}`
    - a subject binding naming `NoDebug.fun/0` in its `expanded_behavior` tier
  when:
    - ExpandedBehavior.hash/2 is called for that MFA
  then:
    - "a `detector_unavailable` finding names reason `:debug_info_stripped` and the module"
    - no hash is returned; no drift finding is emitted for this subject on this tier
  covers:
    - specled.expanded_behavior_tier.no_debug_info_detector_unavailable
    - specled.expanded_behavior_tier.mixed_fixture_coverage
- id: specled.expanded_behavior_tier.scenario.typespec_arg_change_drifts
  given:
    - "a module with `@spec foo(integer()) :: :ok` committed at hash H"
    - "a change to `@spec foo(binary()) :: :ok`"
  when:
    - Typespecs.hash/2 is called
  then:
    - the returned hash differs from H
    - mix spec.check emits `branch_guard_realization_drift` with tier `:typespecs` for `foo/1`
  covers:
    - specled.expanded_behavior_tier.typespecs_hashes_spec_and_type
    - specled.expanded_behavior_tier.typespecs_drift_finding
```
