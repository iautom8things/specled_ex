# Binding Resolution

Resolves a declared MFA binding to a definition, consulting beam first and source
second, with canonicalized AST hashing to detect semantic drift.

## Intent

A `realized_by` entry like `"MyMod.handle_request/3"` points to a function. The guard
needs to answer two questions about that entry per check: does this function exist
(or was it renamed — emit `branch_guard_dangling_binding`), and has its canonicalized
form changed since the last committed hash (emit `branch_guard_realization_drift`).
Beam lookup is authoritative post-compile and sees functions generated by `use`;
source AST lookup is a fallback for when beam is not available.

## Requirements

```yaml spec-requirements
- id: specled.binding.resolve_beam_first
  statement: >-
    SpecLedEx.Realization.Binding.resolve/2 shall look up a target MFA
    on a compiled module via `function_exported?/3` (calling
    `Code.ensure_loaded/1` first) and, on hit, return the AST extracted
    from beam debug_info. Source-file AST lookup is invoked only when
    the beam lookup misses or beam debug_info is not present.
  priority: must
  stability: evolving
- id: specled.binding.resolve_sees_use_generated
  statement: >-
    Binding.resolve/2 shall return `{:ok, ast}` for functions injected
    by `use`-based macros (e.g. DSLs that emit callbacks), because beam
    debug_info reflects post-expansion bytecode that source-only
    traversal would miss.
  priority: must
  stability: evolving
- id: specled.binding.dangling_reported
  statement: >-
    When resolve/2 cannot locate the MFA via beam or source, it shall
    return `{:error, :not_found}` carrying the declared MFA and the
    search paths consulted. ApiBoundary shall translate this into a
    `branch_guard_dangling_binding` finding.
  priority: must
  stability: evolving
- id: specled.binding.canonical_strips_positions
  statement: >-
    SpecLedEx.Realization.Canonical.normalize/1 shall strip line/column
    metadata, sort `@behaviour`/`@derive`/`@spec` lists, and apply
    α-renaming to locally-bound variables whose context is `nil` or
    `Elixir`, so cosmetic refactors (rename, reflow, reordered
    attributes) produce a stable canonical form.
  priority: must
  stability: evolving
- id: specled.binding.canonical_reserved_idents_preserved
  statement: >-
    Canonical.normalize/1's α-rename pass shall NOT rewrite reserved
    identifiers (`__MODULE__`, `__CALLER__`, `__ENV__`, `__DIR__`,
    `__STACKTRACE__`, `__block__`) and shall NOT rewrite variables
    whose context is anything other than `nil` or `Elixir` (macro
    hygiene contexts are left intact).
  priority: must
  stability: evolving
- id: specled.binding.canonical_deterministic_bytes
  statement: >-
    The bytes produced by serializing a canonicalized AST for hashing
    shall be deterministic across runs, OS, and OTP minor versions.
    `:erlang.term_to_binary/2` shall be called with
    `[:deterministic, minor_version: 2]` for every hash input; any call
    site that omits this is a bug.
  priority: must
  stability: evolving
- id: specled.binding.hash_store_atomic
  statement: >-
    SpecLedEx.Realization.HashStore.write/2 shall write
    `.spec/state.json` atomically via `.tmp` + fsync + rename. A crash
    mid-write shall leave the previous committed state intact — partial
    writes are not a reachable state.
  priority: must
  stability: evolving
- id: specled.binding.hasher_version_internal
  statement: >-
    The hasher version shall be carried on hash entries as a module
    attribute `@hasher_version` inside the realization module; it shall
    not be exposed as a user-config key. When a read encounters an
    older `hasher_version`, HashStore shall rehash silently (debug log
    only) and update state. No user-visible finding is produced.
  priority: must
  stability: evolving
```

## Scenarios

```yaml spec-scenarios
- id: specled.binding.scenario.beam_lookup_succeeds
  given:
    - a compiled module `MyMod` defining `handle_request/3`
    - "a binding `\"MyMod.handle_request/3\"`"
  when:
    - Binding.resolve/2 is called with the binding and a Compiler.Context
  then:
    - "the result is `{:ok, ast}` where ast is drawn from beam debug_info"
  covers:
    - specled.binding.resolve_beam_first
- id: specled.binding.scenario.use_generated_function_resolves
  given:
    - a module `PhoenixLikeConsumer` that emits `handle_event/2` via `use SomeDSL`
    - "a binding `\"PhoenixLikeConsumer.handle_event/2\"`"
  when:
    - Binding.resolve/2 is called
  then:
    - "the result is `{:ok, ast}` — source-fallback would miss this function"
  covers:
    - specled.binding.resolve_sees_use_generated
- id: specled.binding.scenario.dangling_binding_detected
  given:
    - "a binding `\"MyMod.missing_fun/0\"` with no matching definition in beam or source"
  when:
    - Binding.resolve/2 is called
  then:
    - "the result is `{:error, :not_found}` carrying the declared MFA"
  covers:
    - specled.binding.dangling_reported
- id: specled.binding.scenario.cosmetic_refactor_stable_hash
  given:
    - "a function body `def f(a, b), do: a + b`"
    - "the same body rewritten as `def f(x, y), do: x + y` with added whitespace and line breaks"
  when:
    - Canonical.normalize/1 is applied to both, then each is hashed
  then:
    - the two hashes are byte-equal
  covers:
    - specled.binding.canonical_strips_positions
    - specled.binding.canonical_deterministic_bytes
- id: specled.binding.scenario.reserved_idents_not_renamed
  given:
    - "a function body that references `__MODULE__` and `__CALLER__`"
  when:
    - Canonical.normalize/1 is applied
  then:
    - "both identifiers appear verbatim in the normalized AST"
  covers:
    - specled.binding.canonical_reserved_idents_preserved
- id: specled.binding.scenario.atomic_write_survives_crash
  given:
    - an existing `.spec/state.json` with committed entries
    - a simulated crash between `.tmp` write and rename (rename is skipped)
  when:
    - HashStore.read/1 is called after the crash
  then:
    - the previous committed state is returned intact
    - no partial `.spec/state.json` is observed
  covers:
    - specled.binding.hash_store_atomic
- id: specled.binding.scenario.hasher_bump_silent_rehash
  given:
    - "a state.json with entries tagged hasher_version: 1"
    - "the module attribute bumped to hasher_version: 2 at compile time"
  when:
    - HashStore.read/1 is called
  then:
    - entries are rehashed internally and written back at version 2
    - no `realization_hasher_version_mismatch` finding is emitted
    - a debug log names the rehash
  covers:
    - specled.binding.hasher_version_internal
```
